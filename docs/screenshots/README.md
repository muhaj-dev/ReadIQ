# Screenshots — how to add them

The main [`README.md`](../../README.md) references the files below. Save each
screenshot you shared into **this folder** (`docs/screenshots/`) using the **exact
filename** so the images render in the README and on GitHub.

The screenshots you provided are composite images (several phone screens per image).
Save each whole composite under its filename — no cropping needed.

| Save as | Which pasted image | What it shows |
|---|---|---|
| `onboarding.png` | The 3-screen onboarding image | Welcome (1/3) · Tell us about you (2/3) · Add your first note (3/3) |
| `home-ask.png` | The 5-screen image with the chat | Home · **Ask AI** (grounded answer + 📌 From your notes tag) · Add Note · Memory · Profile |
| `quiz.png` | The 2-screen image with the MCQ | Home · Quiz (Question 4 of 10, grounded MCQ with green feedback) |
| `memory.png` | The 5-screen "Cell Biology" image | Memory · Note Details · Note Reader · Edit Note · Note Options |
| `add-note.png` | The 5-screen scan/record image | Add Note methods · Scan Page · Scan Result · Record Lecture · Edit Note |
| `deadlines.png` | The 4-screen calendar image | Home · Deadlines calendar · Add Deadline · Reminder picker |
| `profile-settings.png` | The 3-screen dark-profile image | Profile · Settings · Data & Privacy |

## Tips
- Keep the filenames lowercase and exactly as listed.
- PNG or JPG both work; if you use JPG, update the extensions in `README.md` to match.
- To add a brand-new screenshot later: drop the file here and reference it in the
  README's **Screenshots** section with `![caption](./docs/screenshots/<file>.png)`.
